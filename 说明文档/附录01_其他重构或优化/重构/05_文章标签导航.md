# 文章标签导航重构说明文档

## 📌 重构概述

本次重构主要针对文章标签导航栏 (`ArticleNav.vue`) 进行原生化改造和响应式体验优化。通过实现原生 `Tabs` 组件替代 `el-tabs`，修复了 ElementPlus 组件在特定布局下的 Bug，并实现了**移动端吸顶**与**横向滚动交互**。

---

## 🎯 重构原因

### 1. ElementPlus 组件 Bug 与定制困难

**原有问题：**

- **布局 Bug**：原有 `el-tabs` 在切换 `tab-position="top"`（移动端模式）时存在样式错乱和高度计算问题，导致吸顶效果不理想。
- **定制性差**：难以在不同屏幕尺寸下精细控制导航栏的布局（如滚动条隐藏、吸顶联动等）。

### 2. 移动端体验优化需求

**原有问题：**

- **吸顶体验不佳**：旧版导航在移动端无法平滑吸顶，且与下方的文章列表头部 (`ArticleList` 的 `list-order`) 存在定位冲突。
- **视觉干扰**：移动端横向滚动时，原生滚动条长期显示影响美观。

---

## 💡 重构方案

### 核心思路

1.  **原生组件封装**：
    - 创建 `src/components/common/Tabs.vue` 和 `TabItem.vue`，完全接管标签切换逻辑。
    - 实现**横向/竖向**两种布局模式，并在移动端自动隐藏滚动条。

2.  **响应式吸顶架构**：
    - 新建 `ArticleNav.vue`（重构版），原有组件重命名为 `ArticleNavOld.vue`。
    - 使用 `Article.vue` 父级控制 `position: sticky`，避免子组件定位上下文失效。
    - 利用 `ResizeObserver` 动态计算导航栏高度，实时更新 CSS 变量 `--article-nav-height`，确保下方列表紧贴吸顶。

---

## 📝 代码改动详情

### 1. 自定义 Tabs 组件 (src/components/common)

**Tabs.vue**:

- 支持 `direction` 属性切换 `'vertical' | 'horizontal'`。
- **自动隐藏滚动条**：在横向模式下，监听滚动和触摸事件，2秒无操作后自动隐藏滚动条，提升视觉体验。

```typescript
// 滚动条自动隐藏逻辑
const startHideTimer = () => {
  if (props.direction !== 'horizontal') return;
  if (hideTimer) clearTimeout(hideTimer);
  hideTimer = setTimeout(() => {
    isScrollbarVisible.value = false;
  }, 2000);
};
```

### 2. 导航栏重构 (ArticleNav.vue)

- **响应式切换**：通过 `rootStore.isSmallScreen` 自动切换 Tabs 方向。
- **高度联动**：

```typescript
// 使用 ResizeObserver 监听高度变化，更新 CSS 变量
// 供 ArticleList 计算吸顶位置
const updateNavHeightVar = () => {
  if (isSmallScreen.value && navRef.value) {
    const height = navRef.value.offsetHeight;
    document.documentElement.style.setProperty('--article-nav-height', `${height}px`);
  } else {
    document.documentElement.style.setProperty('--article-nav-height', '0px');
  }
};
```

### 3. 吸顶位置精确计算 (ArticleList.vue)

**修改前：**
列表头部吸顶位置固定为导航栏高度。

```css
top: calc(var(--navbarHeight));
```

**修改后：**
动态叠加标签导航的高度（PC端为0，移动端为实际高度）。

```css
/* 完美适配移动端双层吸顶（NavBar + ArticleNav） */
top: calc(var(--navbarHeight) + var(--article-nav-height, 0px));
```

### 4. 父组件布局调整 (Article.vue)

将 Sticky 定位逻辑移至父组件，确保兼容性。

```scss
.article-nav {
  position: sticky;
  top: var(--navbarHeight);
  z-index: 999;
  width: 100%;
}
```

## ❓ Q&A

**Q: 为什么要自己写 Tabs 而不是修复 el-tabs 的样式？**

**A:** ElementPlus 的 `el-tabs` 内部结构复杂，尤其是为了支持 tab-pane 的动态切换，包裹了多层 DOM。在需要实现**移动端横向自动隐藏滚动条**以及**精确的高度联动**时，覆盖 ElementPlus 样式的成本远高于写一个轻量级的原生组件。原生组件仅需百行代码即可完美契合业务需求，且没有额外的包体积负担。
